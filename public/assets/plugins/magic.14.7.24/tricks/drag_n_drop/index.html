Change Log

14.2.14

  • Dropped support MSIE 9 and below for Free Range.

13.5.21

  • Added the optional data-mjf_drag_n_drop_ignore="true" attribute. This will keep elements in the class="dnd_sort_box" container from triggering the drag and drop operation.
  • Dropped elements now retain their child elements much more reliably.

13.4.19

  • data-mjf_drag_n_drop_callback are now passed the newly dropped element and not the element's parent's ID.

13.3.26

  • Better language file loading.

13.3.19

  • Fixed MSIE 10 and touch-based browser Free Range Title Grabs.

12.12.11

  • Completely rewritten using HTML5! This will break your code if you don't update.
  • Sorting Boxes with Add (via click) and Sorting Boxes with Add (via drag in) have been removed. These may or may not be re-added in a future version.
  • Removed the the option to revert draggable elements to their original position. This will be re-added in a future version.
  • Misc bug fixes.

12.8.6

  • Added an option to hide the Remove icon from draggable elements.
  • Added a prompt when removing draggable elements.
  • Added a visual clue that a draggable element is being deleted.
  • Removed draggable elements for the Sorting Boxes with Add (via click) are now added back to their original parent, if they originally had one.
  • Added an option to allow elements to be dragged to any container.
  • Added a localization.
  • Misc code clean up.

12.5.30

  • Fixed a bug where the drag-n-drop containers were being ignored.

12.4.16

  • Fixed a bug where the <body> tag was being assigned the dnd_grabbed CSS class.

11.9.28

  • Production ready

11.9.16

  • Alpha release 2

11.9.6

  • Alpha release 1
